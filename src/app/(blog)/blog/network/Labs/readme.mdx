# Network Labs
计算机网络实验可以在多种平台上进行，这些平台提供了不同的功能和工具，帮助学生和专业人士进行网络实验和模拟。以下是一些常见的计算机网络实验平台：

### 1. Cisco Packet Tracer

**Cisco Packet Tracer** 是由思科公司开发的一款网络模拟工具，广泛用于网络技术的学习和实验。

#### 主要特点：
- **网络模拟**：支持模拟各种网络设备和协议。
- **可视化界面**：提供直观的拖放界面，便于创建和管理网络拓扑。
- **多平台支持**：支持 Windows、macOS 和 Linux。
- **学习资源**：提供丰富的学习资源和实验教程。

#### 适用场景：
- 网络技术学习
- CCNA 等思科认证考试准备
- 网络实验和模拟

### 2. GNS3 (Graphical Network Simulator-3)

**GNS3** 是一个开源的网络模拟器，支持模拟复杂的网络拓扑和设备。

#### 主要特点：
- **开源免费**：完全开源，免费使用。
- **多设备支持**：支持思科、Juniper 等多种网络设备。
- **真实设备集成**：可以与真实网络设备集成，进行混合实验。
- **多平台支持**：支持 Windows、macOS 和 Linux。

#### 适用场景：
- 高级网络实验
- 网络认证考试准备（如 CCNP、CCIE）
- 网络设计和测试

### 3. Mininet

**Mininet** 是一个轻量级的网络模拟器，主要用于模拟软件定义网络（SDN）。

#### 主要特点：
- **轻量级**：占用资源少，适合在普通计算机上运行。
- **SDN 支持**：专为 SDN 设计，支持 OpenFlow 等协议。
- **开源免费**：完全开源，免费使用。
- **脚本化**：支持使用 Python 脚本进行网络配置和管理。

#### 适用场景：
- SDN 研究和实验
- 网络协议开发和测试
- 教学和学习

### 4. EVE-NG (Emulated Virtual Environment Next Generation)

**EVE-NG** 是一个强大的网络仿真平台，支持多种网络设备和虚拟化技术。

#### 主要特点：
- **多设备支持**：支持思科、Juniper、Palo Alto 等多种网络设备。
- **虚拟化支持**：支持 KVM、QEMU 等虚拟化技术。
- **可视化界面**：提供直观的 Web 界面，便于创建和管理网络拓扑。
- **多用户支持**：支持多用户协作和实验。

#### 适用场景：
- 高级网络实验
- 网络认证考试准备（如 CCIE、JNCIE）
- 网络设计和测试

### 5. NetSim

**NetSim** 是一个商业网络模拟器，提供丰富的网络实验和学习资源。

#### 主要特点：
- **商业软件**：需要购买许可证。
- **丰富的实验资源**：提供预定义的实验和学习资源。
- **多设备支持**：支持多种网络设备和协议。
- **可视化界面**：提供直观的拖放界面，便于创建和管理网络拓扑。

#### 适用场景：
- 网络技术学习
- 网络认证考试准备
- 网络实验和模拟

### 6. VirtualBox 和 VMware

**VirtualBox** 和 **VMware** 是两款流行的虚拟化软件，可以用于创建虚拟网络环境。

#### 主要特点：
- **虚拟化支持**：支持创建和管理虚拟机。
- **多操作系统支持**：支持多种操作系统（如 Windows、Linux、macOS）。
- **网络配置**：支持配置虚拟网络和网络适配器。
- **免费和商业版本**：VirtualBox 是开源免费的，VMware 提供免费和商业版本。

#### 适用场景：
- 虚拟网络实验
- 操作系统和网络服务实验
- 教学和学习

`.vdi` 文件是 Oracle VirtualBox 的虚拟磁盘镜像文件格式。VirtualBox 是一个流行的开源虚拟化软件，广泛用于创建和管理虚拟机。你可以使用 VirtualBox 来进行计算机网络实验，通过创建多个虚拟机并配置它们的网络设置来模拟复杂的网络环境。

### VirtualBox 主要特点

1. **虚拟化支持**：支持创建和管理虚拟机，运行多种操作系统（如 Windows、Linux、macOS）。
2. **网络配置**：支持多种网络模式（如 NAT、桥接、内部网络、主机专用网络等），便于模拟不同的网络拓扑。
3. **开源免费**：VirtualBox 是开源软件，免费使用。
4. **快照功能**：支持创建虚拟机快照，便于恢复到之前的状态。
5. **多平台支持**：支持 Windows、macOS 和 Linux。

### 使用 VirtualBox 进行网络实验

以下是使用 VirtualBox 进行网络实验的基本步骤：

#### 1. 安装 VirtualBox

从 [VirtualBox 官方网站](https://www.virtualbox.org/) 下载并安装 VirtualBox。

#### 2. 创建虚拟机

在 VirtualBox 中创建多个虚拟机，每个虚拟机可以运行不同的操作系统。

#### 3. 配置网络

配置虚拟机的网络设置，选择适当的网络模式来模拟不同的网络拓扑。例如：

- **NAT 模式**：虚拟机通过主机的网络连接访问外部网络。
- **桥接模式**：虚拟机直接连接到主机的物理网络，获得独立的 IP 地址。
- **内部网络**：虚拟机之间可以相互通信，但不能访问外部网络。
- **主机专用网络**：虚拟机与主机之间可以通信，但不能访问外部网络。

#### 4. 安装网络工具

在虚拟机中安装所需的网络工具和软件，如 Wireshark、iperf、netcat 等，用于网络分析和测试。

#### 5. 进行网络实验

根据实验需求，配置虚拟机的网络设置，运行网络工具，进行网络实验和测试。

### 示例：配置 VirtualBox 网络实验环境

以下是一个简单的示例，展示如何使用 VirtualBox 配置一个网络实验环境：

#### 1. 创建虚拟机

创建两台虚拟机，分别命名为 `VM1` 和 `VM2`，安装 Ubuntu 操作系统。

#### 2. 配置网络

将两台虚拟机的网络适配器设置为内部网络（Internal Network），网络名称为 `intnet`。

#### 3. 安装网络工具

在两台虚拟机中安装 `iperf` 工具，用于网络性能测试：

```sh
sudo apt-get update
sudo apt-get install iperf
```

#### 4. 进行网络实验

在 `VM1` 上启动 `iperf` 服务器：

```sh
iperf -s
```

在 `VM2` 上运行 `iperf` 客户端，连接到 `VM1` 的 IP 地址，进行网络性能测试：

```sh
iperf -c <VM1_IP_ADDRESS>
```
