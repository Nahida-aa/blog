# 设计文档

## 系统架构
系统采用前后端分离的架构，前端使用React，后端使用Node.js和Express，数据库使用PostgreSQL。

![系统架构图](system_architecture.png)

## 模块设计
### 用户模块
- **功能**：用户注册、登录、认证、管理
- **组件**：
  - 注册组件：处理用户注册逻辑
  - 登录组件：处理用户登录逻辑
  - 认证组件：处理用户认证逻辑
  - 用户管理组件：处理用户信息的管理

### 项目模块
- **功能**：项目创建、编辑、删除、查看
- **组件**：
  - 项目创建组件：处理项目创建逻辑
  - 项目编辑组件：处理项目编辑逻辑
  - 项目删除组件：处理项目删除逻辑
  - 项目查看组件：处理项目查看逻辑

## 数据库设计
### 用户表（users）
| 字段名       | 类型          | 描述           |
| ------------ | ------------- | -------------- |
| id           | SERIAL        | 用户ID         |
| email        | VARCHAR(255)  | 用户邮箱       |
| username     | VARCHAR(255)  | 用户名         |
| password     | VARCHAR(255)  | 用户密码       |
| created_at   | TIMESTAMP     | 创建时间       |
| updated_at   | TIMESTAMP     | 更新时间       |

### 项目表（projects）
| 字段名       | 类型          | 描述           |
| ------------ | ------------- | -------------- |
| id           | SERIAL        | 项目ID         |
| name         | VARCHAR(255)  | 项目名称       |
| description  | TEXT          | 项目描述       |
| owner_id     | INTEGER       | 项目所有者ID   |
| created_at   | TIMESTAMP     | 创建时间       |
| updated_at   | TIMESTAMP     | 更新时间       |

## 接口设计
### 用户注册接口
- **URL**：`/api/register`
- **方法**：POST
- **请求参数**：
  - `email`：用户邮箱
  - `username`：用户名
  - `password`：用户密码
- **响应**：
  - 成功：`{ "message": "注册成功" }`
  - 失败：`{ "error": "注册失败" }`

### 用户登录接口
- **URL**：`/api/login`
- **方法**：POST
- **请求参数**：
  - `email`：用户邮箱
  - `password`：用户密码
- **响应**：
  - 成功：`{ "token": "JWT_TOKEN" }`
  - 失败：`{ "error": "登录失败" }`
