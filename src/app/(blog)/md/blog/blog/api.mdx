- **创建仓库**：
  ```http
  POST /user/repos
  ```

- **删除仓库**：
  ```http
  DELETE /repos/{owner}/{repo}
  ```

- **获取仓库内容**：
  ```http
  GET /repos/{owner}/{repo}/contents/{path}
  ```

### 2. 分支相关 API

- **获取仓库的分支**：
  ```http
  GET /repos/{owner}/{repo}/branches
  ```

- **获取分支信息**：
  ```http
  GET /repos/{owner}/{repo}/branches/{branch}
  ```

### 3. 提交相关 API

- **获取仓库的提交**：
  ```http
  GET /repos/{owner}/{repo}/commits
  ```

- **获取单个提交**：
  ```http
  GET /repos/{owner}/{repo}/commits/{sha}
  ```

### 4. 拉取请求（Pull Requests）相关 API

- **获取仓库的拉取请求**：
  ```http
  GET /repos/{owner}/{repo}/pulls
  ```

- **创建拉取请求**：
  ```http
  POST /repos/{owner}/{repo}/pulls
  ```

- **合并拉取请求**：
  ```http
  PUT /repos/{owner}/{repo}/pulls/{pull_number}/merge
  ```

### 5. 问题（Issues）相关 API

- **获取仓库的问题**：
  ```http
  GET /repos/{owner}/{repo}/issues
  ```

- **创建问题**：
  ```http
  POST /repos/{owner}/{repo}/issues
  ```

- **更新问题**：
  ```http
  PATCH /repos/{owner}/{repo}/issues/{issue_number}
  ```

### 6. 用户相关 API

- **获取用户信息**：
  ```http
  GET /users/{username}
  ```
```sh
GET /users/{username}/repos
```
- **获取当前认证用户的信息**：
  ```http
  GET /user
  ```

### 7. 组织相关 API

- **获取组织信息**：
  ```http
  GET /orgs/{org}
  ```

- **获取组织的成员**：
  ```http
  GET /orgs/{org}/members
  ```

### 示例

以下是一个使用 REST Client 插件的示例，展示了如何获取仓库的内容：

```rest
@baseUrl = https://api.github.com
@owner = Nahida-aa
@repo = blog
@path = 
@token = 

GET {{baseUrl}}/repos/{{owner}}/{{repo}}/contents/{{path}}
Authorization: Bearer {{token}}
```

通过这些 API，你可以与 GitHub 平台进行各种交互，如管理仓库、处理拉取请求、管理问题等。详细的 API 文档可以参考 [GitHub API 文档](https://docs.github.com/en/rest)。
